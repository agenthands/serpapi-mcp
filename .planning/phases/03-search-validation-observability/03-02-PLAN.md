  <action>
1. Create `internal/search/validation.go` with three exported validation functions:

   a. `ValidateEngine(engine string) error`:
      - Call `engines.EngineNames()` to get valid engine list
      - Check if `engine` is in the list (use sort.SearchStrings since EngineNames returns sorted)
      - If not found, return error with message: `invalid_engine: engine '{engine}' not found. Available engines: {comma-separated sorted list}`
      - If found, return nil

   b. `ValidateMode(mode string) error`:
      - Check if mode is "complete" or "compact" (D-10)
      - If not, return error with message: `invalid_mode: mode must be 'complete' or 'compact', got '{mode}'`
      - If valid, return nil

   c. `ValidateRequiredParams(engine string, params map[string]any) error`:
      - Call `engines.RequiredParams(engine)` to get required param names
      - If engine not found (returns nil), skip validation (will be caught by ValidateEngine)
      - Check each required param name exists as a key in `params`
      - Collect missing param names
      - If any missing, return error with message: `missing_params: missing required parameter(s) {comma-separated names} for engine '{engine}'`
      - If all present, return nil

2. Create `internal/search/validation_test.go` with test coverage for all 6 validation behaviors above.

3. Update `internal/search/search.go` to integrate validation into the search tool handler. Add validation calls at the TOP of the handler function, BEFORE the SerpApi HTTP call:
   - After extracting engine name from params: call `ValidateEngine(engine)` → if error, return `toolError("invalid_engine", err.Error())`
   - After extracting mode: call `ValidateMode(mode)` → if error, return `toolError("invalid_mode", err.Error())`
   - After engine validation passes, extract params map and call `ValidateRequiredParams(engine, paramsMap)` → if error, return `toolError("missing_params", err.Error())`
   - ORDER: engine validation → mode validation → required params validation → SerpApi HTTP call
   - Do NOT make HTTP calls if any validation fails — this ensures validation errors are fast and don't waste SerpApi quota

4. Add structured logging in the search handler:
   - Log at INFO level: `slog.Info("search request", "engine", engine, "mode", mode, "params_count", len(paramsMap))`
   - Log at ERROR level on validation failures: `slog.Error("search validation failed", "error", err, "engine", engine, "mode", mode)`
   - Note: correlation ID logging will be added in Task 2 when the middleware is wired
  </action>

   - Implement `CorrelationIDMiddleware(next http.Handler) http.Handler`:
     a. Check if `X-Correlation-ID` header exists in request
     b. If exists, use that value as correlation ID (allows client-provided tracing)
     c. If not, generate new ID using `crypto/rand`: read 16 bytes, hex-encode to 32-char string
     d. Store in request context via `context.WithValue(r.Context(), correlationIDKey, id)`
     e. Set `X-Correlation-ID` response header to the correlation ID
     f. Call `next.ServeHTTP(w, r.WithContext(ctx))`
