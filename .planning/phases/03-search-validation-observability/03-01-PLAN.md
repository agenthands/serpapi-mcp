  <action>
1. Add exported accessor functions to `internal/engines/engines.go` so the search package can access engine data for validation (will be used in Plan 02 but needed structurally now):
   - Add a package-level `var engineNamesStore []string` and `var schemasStore map[string]*engineSchema` to cache loaded data
   - In `LoadAndRegister`, populate these stores after loading: `engineNamesStore = engineNames` and `schemasStore = schemas`
   - Add `func EngineNames() []string` that returns a copy of `engineNamesStore`
   - Add `func RequiredParams(engineName string) []string` that iterates `schemasStore[engineName].Raw`, extracts params with `"required": true`, returns the param names. Parse the `params` object from the raw JSON, iterate each key, check if the object has `"required": true`.

2. Implement `internal/search/search.go`:
   - Define `SearchToolInput` struct with `Params map[string]any` and `Mode string` fields (for explicit documentation, though we use raw schema for registration).
   - Define error types as constants: `errInvalidEngine`, `errInvalidMode`, `errMissingParams`, `errRateLimited`, `errInvalidAPIKey`, `errForbidden`, `errSearchError`, `errMissingAPIKey`.
   - Define `toolError(code, message string) *mcp.CallToolResult` helper that creates a `CallToolResult` with `IsError: true` and `Content` containing a `TextContent` with JSON body `{"error": code, "message": message}`. This implements D-01 and D-02.
   - Define `RegisterSearchTool(srv *mcp.Server, logger *slog.Logger)` that calls `srv.AddTool` with:
     - Tool definition: name "search", description matching Python server's docstring, and `InputSchema` as `json.RawMessage` with `{"type":"object","properties":{"params":{"type":"object","description":"Dictionary of engine-specific parameters. Common: q (query), engine, location, num"},"mode":{"type":"string","enum":["complete","compact"],"default":"complete","description":"Response mode: complete returns full JSON, compact removes metadata fields"}},"required":["params"]}`
     - Handler function that:
       a. Unmarshals `req.Params.Arguments` into a struct with `Params map[string]any` and `Mode string`
       b. Sets default mode to "complete" if empty (D-05)
       c. Extracts API key from `server.APIKeyFromContext(req.Context())` (D-06) — if empty, return `toolError("missing_api_key", "...")`
       d. Extracts `engine` from params, defaults to "google_light" (D-05, SRCH-02)
       e. Constructs SerpApi URL using `url.URL` and `url.Values`: base `https://serpapi.com/search`, add `api_key`, `engine`, and all params as query parameters
       f. Makes GET request via `&http.Client{Timeout: 30 * time.Second}` (D-07)
       g. Checks response status code:
          - 200: continue to process body
          - 429: return `toolError("rate_limited", "Rate limit exceeded. Please try again later.")` (D-03, SRCH-07)
          - 401: return `toolError("invalid_api_key", "Invalid SerpApi API key. Check your API key in the path or Authorization header.")` (D-03, SRCH-07)
          - 403: return `toolError("forbidden", "SerpApi API key forbidden. Verify your subscription and key validity.")` (D-03, SRCH-07)
          - 5xx/other: return `toolError("search_error", fmt.Sprintf("SerpApi returned status %d: %s", resp.StatusCode, body))` (D-03, SRCH-07)
       h. Reads response body, parses as `map[string]any`
       i. If mode is "compact" (D-08): delete keys `"search_metadata"`, `"search_parameters"`, `"search_information"`, `"pagination"`, `"serpapi_pagination"` from the map
       j. Marshal result back to JSON, return as `TextContent` in `CallToolResult{Content: []mcp.Content{&mcp.TextContent{Text: resultJSON}}, IsError: false}`
       k. If SerpApi returns non-object JSON (e.g. array), return it as-is without compact processing
       l. Wrap entire handler in error recovery: any unexpected error returns `toolError("search_error", err.Error())`

3. Test setup: use `httptest.NewServer` to mock SerpApi responses:
   - Create a test HTTP server that returns different status codes based on query parameters (e.g., `?_test_status=429`)
   - In tests, override the SerpApi base URL via a package-level variable `serpapiBaseURL` (default `"https://serpapi.com/search"`, overridden in tests)
   - Do NOT use environment variables for the base URL override — use a package-level `var` that tests can modify

4. Write `internal/search/search_test.go` covering all 7 behaviors listed in `<behavior>` above.
