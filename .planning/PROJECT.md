# SerpApi MCP Server

## What This Is

A Model Context Protocol (MCP) server that exposes SerpApi's multi-engine search capabilities to AI agents. It runs as a streamable HTTP service, authenticates via API key (path or header), and provides a single unified `search` tool with per-engine parameter schemas exposed as MCP resources. Deployed as a hosted service at mcp.serpapi.com and also self-hostable via Docker or uvicorn.

## Core Value

AI agents can search any SerpApi-supported engine (Google, Bing, DuckDuckGo, YouTube, etc.) through a single, authenticated MCP endpoint with structured parameter discovery.

## Context

- **Existing codebase**: Single-file Python MCP server (`src/server.py`) using FastMCP on Starlette + uvicorn
- **Engine schemas**: 100+ engines auto-generated by `build-engines.py` scraping SerpApi playground, stored as JSON in `engines/`
- **Deployment**: AWS Copilot, pushes to main trigger deployment (`copilot svc deploy --name mcp-server-api`)
- **CI**: Format check only (`uv format --check`) on PRs
- **No test suite exists**: `pytest` is a dev dependency but no tests are written
- **No type annotations**: `mypy` config requires `disallow_untyped_defs` but server.py has no annotations

## Constraints

- **Python**: 3.13+ required (`requires-python >= 3.12`, `.python-version` is 3.13)
- **Package manager**: uv (not pip)
- **Style**: black (line-length 100) + isort (profile=black)
- **Type checking**: mypy with `disallow_untyped_defs = true`
- **Generated code**: `engines/*.json` must not be hand-edited; regenerate via `python build-engines.py`
- **Transport**: Streamable HTTP only (no SSE/WebSocket)
- **Auth model**: API key in URL path or Authorization header, validated by `ApiKeyMiddleware`

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| FastMCP on Starlette | Standard Python MCP framework with HTTP transport support | ✓ Good |
| Single `search` tool | Simpler API surface — engine selection via `params.engine` | ✓ Good |
| Path-based auth (`/{KEY}/mcp`) | Matches Smithery convention, easiest for MCP clients | ✓ Good |
| Engine schemas as MCP resources | Enables dynamic parameter discovery by AI agents | ✓ Good |
| `build-engines.py` code generation | Keeps engine data in sync with SerpApi playground automatically | ✓ Good |
