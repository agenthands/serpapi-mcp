# AGENTS.md

## Commands

```bash
uv sync                              # Install dependencies
uv run src/server.py                 # Run the server locally (default port 8000)
python build-engines.py              # Regenerate engines/*.json from SerpApi playground (requires network)
uv format --check                    # CI format check (used in PR workflow)
uv format                            # Auto-format code
```

## Architecture

Single-package Python MCP server. Everything lives in `src/server.py`.

- **Entry point**: `src/server.py:main()` → `serpapi-mcp` script in pyproject.toml
- **Framework**: FastMCP on Starlette + uvicorn, streamable HTTP transport
- **Auth**: API key extracted from URL path (`/{KEY}/mcp`) or `Authorization: Bearer` header, handled by `ApiKeyMiddleware`
- **Engine schemas**: `engines/*.json` are generated by `build-engines.py` (scrapes the SerpApi playground) and loaded at startup as MCP resources at `serpapi://engines` and `serpapi://engines/<engine>`
- **Search tool**: Single `search` tool; all engines and parameters go through `params` dict; default engine is `google_light`

## Constraints

- Python 3.13+ required (`requires-python = ">=3.12"`, `.python-version` is 3.13)
- `engines/*.json` is generated code — do not hand-edit; run `python build-engines.py` instead
- `ENGINES_DIR` is resolved relative to `src/server.py` (`Path(__file__).resolve().parents[1] / "engines"`), so it points to the repo-root `engines/` directory
- Deployment is via AWS Copilot (`copilot svc deploy --name mcp-server-api`), triggered on push to main
- Env vars: `MCP_HOST` (default `0.0.0.0`), `MCP_PORT` (default `8000`), `SERPAPI_API_KEY` (not required via env when using path/header auth)
- No test suite exists in the repo despite `pytest` being a dev dependency
- `mypy` config requires `disallow_untyped_defs = true` — add type annotations to all function definitions
- `black` line length is 100, `isort` profile is `black`
